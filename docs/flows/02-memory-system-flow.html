Large Language Models are stateless - they forget everything after each response. Memory systems solve this fundamental limitation.

LAYER 2

Long-Term Memory

Persistent memory across sessions for learning and recall

JSON Files
Duration: Permanent

üìù Memory Entries

Structured records of conversations, decisions, code changes, errors, and solutions with importance scoring.

üè∑Ô∏è Tagging System

Categorize memories with tags for efficient retrieval. Find related past experiences quickly.

üìä Importance Scoring

1-5 rating system. Critical bug fixes score 5, routine operations score 1. Prioritizes recall.

üîê Project Isolation

Each project has separate memory storage using MD5 hash. No cross-project contamination.

LAYER 3

Dynamic context selection based on relevance and token budget

Computed On-Demand
Duration: Per Request

üéØ File Prioritization

Score files by extension (.py=0.9, .md=0.5) and relevance to query. Most important first.

üìè Token Budget

Estimate tokens (chars √ó 0.25) and fill context window without overflow. Typically 32K limit.

‚≠ê Important Files

Auto-include config files: package.json, pyproject.toml, README.md regardless of query.

üîÑ Dynamic Refresh

Context rebuilt for each request based on current query and workspace state.

üéØ Interview Key Points

Q: Why three layers of memory?

Each layer serves a different purpose: Session (fast, temporary), Long-term (persistent learning), Smart Context (dynamic relevance). Like CPU cache, RAM, and disk storage.

Q: How do you handle token limits?

Smart Context Window estimates tokens (chars √ó 0.25), prioritizes by file importance, and fills the budget. Most relevant content first, truncate if needed.

Q: What's the importance scoring for?

Not all memories are equal. A critical bug fix (score 5) should be recalled more readily than a routine file read (score 1). It's prioritized relevance.

Q: How is project isolation achieved?

Each project gets a unique ID via MD5 hash of the workspace path. Memories are stored in separate files under data/memory/{project_id}/.
