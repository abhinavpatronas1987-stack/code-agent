User Input

User enters a message through the CLI interface or sends a request to the REST API. The input can be natural language, slash commands, or file attachments.

Input Parsing

The input is parsed to detect slash commands (/git, /test) vs natural language. Commands are routed directly to handlers, while natural language goes to the agent.

if message.startswith("/"): handle_command(message) else: agent.run(message)
Parser Command Router

Context Loading

Smart Context Window loads relevant files, memory entries, and attachments. Prioritizes by file type and importance while staying within token budget.

context = SmartContextWindow(budget=32000) context.add_relevant_files(query, workspace) context.add_memory_entries(recall(query)) context.add_attachments(session.attachments)

Agent Orchestration

CodingAgent receives the enriched request. Agno framework manages conversation state, system prompt injection, and prepares the LLM call.

response = agent.run( message=user_input, context=loaded_context, stream=True )

LLM Processing

The LLM (Claude or Ollama) processes the request. It generates a response which may include tool calls (function calling) for actions like reading files.

LLM Output: { "content": "Let me read the file first.", "tool_calls": [{ "name": "read_file", "arguments": {"path": "auth.py"} }] }
Claude Ollama Function Calling

Tool Execution

Detected tool calls are routed to their implementations. Tools execute in a sandboxed environment with logging. Results are captured and returned.

for tool_call in response.tool_calls: result = execute_tool( name=tool_call.name, args=tool_call.arguments ) log_tool_execution(tool_call, result) feed_result_to_llm(result)
Tool Router Sandbox
üì°

Response Streaming

The LLM generates the final response incorporating tool results. Response is streamed token-by-token to the user for immediate feedback.

for chunk in response.stream(): if chunk.content: display(chunk.content) # ~100ms first token if chunk.tool_status: show_tool_progress(chunk.tool_status)
Streaming Real-time

Memory Recording

After completion, the interaction is recorded. Session memory updates in SQLite, long-term memory entries are created for important events, and checkpoints are saved.

session.add_message(user_input, response) memory.record_entry( type="conversation", content=summary, importance=calculate_importance() ) checkpoint.auto_save()
SQLite Long-term Memory Checkpoint

‚ö° Performance Optimizations

  • Streaming reduces perceived latency to ~100ms
  • Smart context prevents token overflow
  • Lazy loading of non-essential components
  • Caching via @lru_cache for settings
  • Parallel tool execution where possible

üîí Security Checkpoints

  • Working directory scoping enforced
  • Command timeout (120s default)
  • Path validation and normalization
  • Sensitive file pattern detection
  • Confirmation prompts for destructive ops

üîÑ Error Handling

  • Tool errors captured and fed back to LLM
  • Retry logic with exponential backoff
  • Graceful degradation on LLM failure
  • Checkpoint restoration on critical errors
  • User notification for recoverable issues
